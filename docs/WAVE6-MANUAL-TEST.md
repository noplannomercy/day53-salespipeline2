# Wave 6 수동 테스트 가이드

```bash
npm run dev   # http://localhost:3000
```

---

## W6-1 딜 기한 알림 자동 생성

**목적**: 마감일이 D-3일, D-1일인 딜에 알림이 자동 생성되는지 확인

1. 딜 목록(`/deals`) 또는 칸반(`/kanban`) 페이지 접속
2. 헤더 우측 🔔 벨 아이콘 클릭
3. **기대 결과**: 마감일 임박 딜의 "마감 임박 (D-N): [딜 제목]" 알림 목록 확인

> 시드 데이터에 마감일 임박 딜이 없으면: `/deals` → 딜 생성 → 마감일을 오늘 + 1일 또는 + 3일로 설정 → 페이지 새로고침 후 벨 아이콘 확인

## W6-2 딜 상세 내 복제 버튼

**목적**: 딜 상세 페이지에서 복제 버튼으로 새 딜 생성 및 이동

1. `/deals` → 임의 딜 클릭하여 상세 페이지 진입
2. 페이지 상단 헤더에서 **"딜 복제"** 버튼 확인
3. 버튼 클릭
4. **기대 결과**:
   - 버튼이 "복제 중..." 로딩 텍스트로 전환됨
   - 복제된 새 딜의 상세 페이지(`/deals/[newId]`)로 자동 이동
   - 새 딜 제목이 "[원본 딜명] (복사본)" 형식

## W6-3/W6-4 이메일 템플릿

**목적**: 이메일 작성 시 템플릿 선택으로 본문 자동 채우기

### 템플릿 생성 (사전 준비)
> 현재 템플릿 관리 페이지는 별도 미구현 — EmailForm 내에서 직접 확인

### 이메일 작성 테스트
1. `/emails` → **"이메일 작성"** 버튼 클릭
2. 폼 상단에 **템플릿 선택 드롭다운** 확인
3. 드롭다운 클릭 → 저장된 템플릿 목록 표시 확인
4. 템플릿 선택
5. **기대 결과**: 제목(subject)과 본문(body) 필드가 템플릿 내용으로 자동 채워짐

> 시드 템플릿이 없으면 드롭다운이 숨겨짐 — 정상 동작

## W6-5 리포트 확장 (멤버 성과 + 전환 퍼널)

**목적**: 리포트 페이지에 2개 탭이 추가됐는지 확인

1. `/reports` 접속
2. 탭 목록 확인 — 총 **7개 탭** 있어야 함:
   - 파이프라인 현황 / 매출 실적 / 활동 현황 / 매출 예측 / 리드 출처 / **멤버 성과** / **전환 퍼널**

### 멤버 성과 탭
3. **"멤버 성과"** 탭 클릭
4. **기대 결과**:
   - 멤버별 딜 성사율 바 차트
   - 하단 요약 테이블 (멤버명 / 성사 딜 수 / 전체 딜 수 / 성사율 / 활동 수)

### 전환 퍼널 탭
5. **"전환 퍼널"** 탭 클릭
6. **기대 결과**:
   - 스테이지별 딜 수 수평 바 차트
   - 하단 요약 테이블 (스테이지명 / 딜 수 / 총 금액)
